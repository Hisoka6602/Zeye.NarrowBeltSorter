# 落格精准度动态成熟延迟模型改造清单（最小侵入）

## 1. 目标与边界

- 目标：在不重构现有“主协调 + 上车 + 落格”服务边界的前提下，降低包裹成熟时间抖动导致的落格偏差。
- 边界：优先使用配置化 + 轻量模型 + 在线校正，不改动核心设备驱动协议流程，不新增高侵入链路。
- 时间语义：全链路统一使用本地时间（Local Time），不引入 UTC 转换。

## 2. 问题拆解（结合现场约束）

包裹成熟时机不稳定，通常来自以下叠加因素：

1. **包裹长度差异客观存在，但当前无法实时测量长度**：长度显式补偿暂不可直接使用。
2. **线速度波动**：变频、负载变化会导致瞬时速度偏离标称速度，当前仅可依赖感应小车变化事件估算。
3. **振动扰动**：输送带接缝、过渡段、机架共振会造成相位抖动（提前/延迟），短期无法从物理侧消除。
4. **摩擦系数差异**：包装材质不同导致打滑与粘滞差异，但当前无法实时提供材质特征。

因此，成熟延迟不应是单一固定常量，应采用“**基础值 + 可观测动态项 + 在线残差校正**”。

## 3. 最小侵入模型设计

## 3.1 核心公式

```text
成熟延迟(ms) = 基础延迟 + 速度补偿 + 相位扰动补偿 + 残差校正
```

其中：

- `基础延迟`：原有固定值（现网可直接复用）。
- `速度补偿`：依据事件估算实时速度与参考速度偏差补偿。
- `相位扰动补偿`：依据振动风险区段/相位桶进行小幅附加补偿。
- `残差校正`：用最近 N 个样本误差做低权重 EMA 更新，缓慢修正模型偏置。

## 3.2 推荐实现表达式

```text
delay = baseMs
      + kVel * (refSpeedMmps - currentSpeedMmps)
      + phaseBiasMs
      + emaResidualMs
```

并执行边界钳制：

```text
delay = clamp(delay, minDelayMs, maxDelayMs)
```

> 说明：速度项使用 `(refSpeed - currentSpeed)` 便于表达“速度慢则延迟增加，速度快则延迟减小”；长度与材质不可观测影响统一通过残差校正缓慢吸收。

## 4. 需要新增的配置项（建议）

建议新增 `Sorting:Timing:DynamicMaturity` 节，字段保持中文注释：

- `Enabled`：是否启用动态成熟模型
- `BaseDelayMs`：基础成熟延迟
- `ReferenceSpeedMmps`：参考输送速度
- `SpeedCompensationGain`：速度补偿增益 `kVel`
- `PhaseCompensationBucketCount`：相位补偿桶数量（如 16/32）
- `PhaseCompensationGain`：相位补偿学习增益
- `ResidualEmaAlpha`：残差 EMA 系数（建议 0.02~0.10）
- `MinDelayMs`：最小延迟边界
- `MaxDelayMs`：最大延迟边界
- `SampleWindowSize`：残差样本窗口大小

## 5. 按现有架构的最小落地点

1. **主协调服务（SortingTaskOrchestrationService）**
   - 保留成熟泵送流程，仅把固定成熟延迟替换为“动态成熟延迟计算结果”。
   - 同一事件链路只采样一次本地时间并透传，避免重复取时抖动。

2. **上车服务（SortingTaskCarrierLoadingService）**
   - 不改变核心职责，仅消费已成熟包裹。
   - 可记录“预测成熟时刻 vs 实际上车时刻”误差样本，用于残差 EMA。

3. **落格服务（SortingTaskDropOrchestrationService）**
   - 不改落格主流程，仅利用既有命中窗口配置兜底消化小幅误差。

## 6. 现场工程化步骤（建议）

1. **阶段A：只上线观测，不闭环**
   - 打日志输出 `base/vel/phase/residual/finalDelay` 分量。
   - 连续采样 1~2 班次，确认延迟分布是否收敛。

2. **阶段B：启用速度补偿**
   - 固定 `emaResidualMs=0`，先校准 `kVel`。
   - 观察落格偏差是否从“单侧偏移”转为“零均值小波动”。

3. **阶段C：启用残差 EMA 微调**
   - 打开低权重 EMA，限制每次更新幅度，防止过拟合。
   - 对异常大偏差样本做剔除（如通信抖动、急停恢复后的前若干件）。

4. **阶段D：相位扰动补偿**
   - 在已知振动区段/相位桶引入额外 `phaseBiasMs`，先小步长上线。

## 7. 关键防抖与稳健策略

- **限幅**：所有补偿项和最终 delay 必须限幅。
- **异常样本剔除**：超阈值残差不入 EMA。
- **冷启动保护**：样本不足时仅使用 `base + 轻量补偿`。
- **状态切换保护**：急停/复位后前 N 件仅观测不学习。
- **事件估速去抖**：感应事件间隔先做去极值再做平滑，避免瞬时尖峰直接驱动成熟延迟。

## 8. 验收清单（Checklist）

- [ ] 动态模型输出可观测（各分量日志齐全）
- [ ] 延迟结果已做最小/最大边界钳制
- [ ] 速度补偿参数可配置且支持热更新（如现有架构允许）
- [ ] 残差 EMA 具备异常样本剔除与冷启动保护
- [ ] 相位扰动补偿可独立开关
- [ ] 现场对比显示落格偏差 P95 / P99 明显下降

## 9. 后续可完善点

1. 将相位桶模型升级为“分区段 + 分速度段”联合模型（不同线段不同增益）。
2. 条件允许时接入包裹长度/材质识别特征，形成显式补偿项并降低残差学习压力。
3. 与视觉或高精度编码器数据融合，进一步压缩 P99 尾部误差。
