# 多从站稳速难题分析与工程解决方案

> 适用场景：雷码 LM1000H 变频器多从站（Multi-Slave Modbus RTU/TCP）闭环稳速控制  
> 关联实现：`Zeye.NarrowBeltSorter.Drivers/Vendors/LeiMa/LeiMaLoopTrackManager.cs`  
> 关联算法：`Zeye.NarrowBeltSorter.Core/Algorithms/PidController.cs`

---

## 1. 问题背景

在单从站场景下，PID 闭环稳速通常可以较快收敛并保持稳定。但当扩展到多从站时——即多台变频器驱动同一条皮带或环形轨道——往往出现以下典型问题：

- 速度持续波动，始终无法进入稳定窗口  
- 收敛速度远慢于单机场景，或根本无法收敛  
- 各从站速度之间出现"此消彼长"的相互干扰  
- PID 参数需要大幅调小才能维持不振荡，但随之而来是响应迟缓  

本文系统分析上述现象的根本原因，并对比工业界常见解决方案与产品案例，结合本仓库当前架构给出改进建议。

---

## 2. 单从站易稳速的本质原因

单从站闭环系统具备以下有利特性：

| 特性 | 描述 |
|---|---|
| 唯一反馈源 | 只有一个编码器读数进入 PID，反馈无歧义 |
| 唯一执行器 | PID 输出直接对应一台变频器，无"命令-响应"分叉 |
| 通信延时确定 | 单次 Modbus 请求响应时间固定，系统纯滞后可准确建模 |
| 负载耦合简单 | 机械结构单一，负载变化可被单 PID 直接跟踪补偿 |
| PID 参数收敛快 | 被控对象简单，Ziegler-Nichols 等整定方法容易奏效 |

## 3. 多从站难以稳速的根本原因

### 3.1 顺序 Modbus 轮询引入的采样时差

**当前代码逻辑（`LeiMaLoopTrackManager.cs` 第 707–720 行）：**

```csharp
foreach (var (slaveAddress, adapter) in _slaveClients) {
    // 逐个从站串行读取编码器速度
    var (sampleOk, sampleRaw) = await _safeExecutor.ExecuteAsync(...);
    samples.Add((slaveAddress, LeiMaSpeedConverter.HzToMmps(...)));
}
```

RS-485 总线是半双工介质，同一时刻只允许一个主站帧在线，因此 N 个从站**必须串行轮询**。

> **假设前提**：以下表格与计算均以单从站典型正常响应时间 `T_slave = 50 ms` 为基准（实测值因 RS-485 波特率、线缆长度与变频器固件差异而不同，`TimeoutMilliseconds` 是异常超时阈值，不等于正常响应时间）。

| 从站数量 | T_slave（典型） | 总采样耗时 | PID 纯滞后增量 |
|---|---|---|---|
| 1 | T_slave = 50 ms | 50 ms | 0 ms |
| 2 | T_slave = 50 ms | 100 ms | 50 ms |
| 3 | T_slave = 50 ms | 150 ms | 100 ms |
| N | T_slave = 50 ms | N×T_slave | (N-1)×T_slave |

**影响：**  
- 采样到的第 1 个从站数据比第 N 个从站数据早约 `(N-1) × T_slave`（以 T_slave = 50ms 为例，3 从站时差达 100ms），导致聚合速度是**不同时刻**的拼接，违背 PID 对同步反馈的假设。  
- PID 的 `SamplePeriodSeconds` 设置为轮询间隔（默认 300ms），但实际控制周期 = 轮询间隔 + N × T_slave，积分和微分项的时域计算存在**系统性偏差**。  
- 纯滞后（Dead Time）增大是 PID 稳定性的最大威胁：经典稳定性条件 `Kp × L < 常数` 要求随 L（滞后）增大必须同比降低 Kp，否则系统振荡。

### 3.2 皮带机械耦合引入的扰动串扰

多台变频器驱动同一条皮带时，皮带张力将所有驱动节点机械耦合在一起。

```
[从站1电机] ---皮带张力--- [从站2电机] ---皮带张力--- [从站3电机]
```

**耦合效应：**  
- 任意一台变频器的转矩变化，会通过皮带拉力立即影响其他节点的实际速度。  
- PID 输出指向所有从站的同一转矩值，但皮带张力在各节点的分布因包络角、皮带弹性、滑轮半径等物理参数不均匀，实际传动效率和负载各不相同。  
- 当某处皮带承重（如包裹压在两驱动节点之间时），该段张力骤增，两侧驱动节点感知到的反力瞬时变化，导致速度反馈阶跃，PID 过度补偿。

### 3.3 统一转矩给定无法补偿个体负载差异

**当前代码逻辑（`ExecutePidClosedLoopAsync` 步骤3）：**

```csharp
await WriteRegisterToAllSlavesAsync(
    LeiMaRegisters.TorqueSetpoint,
    torqueRaw,               // 所有从站写入相同的 torqueRaw
    "LeiMa.PidClosedLoop.WriteTorqueSetpoint",
    cancellationToken);
```

单 PID 控制器对全部从站输出**同一个转矩给定值**。但各从站所在位置的机械阻力（弯道阻力、皮带摩擦差异、包裹分布）存在差异，同一转矩在不同位置产生的实际速度不同：

```
相同 P3.10 = 800 → 从站1（直段，轻载）速度 = 1620 mm/s
                  → 从站2（弯道，重载）速度 = 1480 mm/s  ← 偏慢 140 mm/s
```

PID 的聚合反馈（Min/Avg/Median）无法区分这两种情况，只能整体补偿，导致轻载从站超速而重载从站仍欠速，或来回振荡。

### 3.4 速度聚合策略引入的系统偏差

**当前支持的三种策略（`AggregateSpeed` 方法）：**

| 策略 | 行为 | 稳速影响 |
|---|---|---|
| `Min`（最小值） | PID 始终以最慢从站为基准补偿 | 其他从站被拉至超速，总体偏高 |
| `Avg`（均值） | 快慢从站速度互相抵消 | 局部欠速被掩盖，单从站可能严重偏差 |
| `Median`（中位数） | 抗异常值能力强 | 奇数从站时性质最好，但偶数时为两中间值均值，仍有信息损失 |

任何单一聚合策略都无法同时保证所有从站速度一致，且将多个噪声源混合后，PID 看到的是**复合噪声**而非单一传感器噪声，增益余量进一步压缩。

### 3.5 噪声放大效应

- 每个从站编码器都有独立的测量噪声（量化误差、机械抖动、电磁干扰）。  
- 使用 Min 策略时，每帧选出的是 N 个随机噪声中最小的一个，这是**极值分布**，其方差远大于单次测量的方差。  
- 等效于 PID 的反馈信号噪声随从站数量增多而增大，迫使降低 Kd（微分项对噪声最敏感），进一步削弱抗扰动能力。

### 3.6 PID 参数整定难度非线性增加

以上五个因素相互耦合，任何一个参数变化都会同时影响多个维度：

```
增大 Kp → 响应更快 → 但放大了噪声 + 纯滞后的振荡风险
增大 Ki → 消除稳态误差 → 但延长了从站间同步收敛时间
增大 Kd → 抑制超调  → 但放大了多从站聚合噪声
```

**结论：** 单从站场景中可以良好工作的 PID 参数，在多从站场景中面临更大的纯滞后、更强的耦合扰动与更高的反馈噪声，其稳定裕度大幅缩小，往往需要将参数降至接近纯开环的保守值，从而牺牲了闭环的本质优势。

## 4. 工业界主流解决方案

### 4.1 主从转矩跟随（Master-Slave Torque Following）

**原理：**  
指定一台变频器为"主机"，运行速度 PID 闭环；其余变频器为"从机"，直接复制主机的转矩输出，无需独立 PID。

```
主机：速度 PID → 转矩给定 T_master
从机1：直接写入 T_master（或 T_master × 负载比例）
从机2：直接写入 T_master（或 T_master × 负载比例）
```

**优点：**  
- 从机无延时反馈，彻底消除从机侧的 PID 失稳问题。  
- 皮带张力自然均衡（相同转矩 → 均等分担）。  
- 参数整定只需整定主机 PID，简单高效。

**缺点：**  
- 从机无独立反馈，若皮带局部打滑，从机无法感知。  
- 需要驱动器支持转矩跟随模式（或上位机实时广播转矩）。

**代表产品：**  
- 西门子 SINAMICS S120（CU320-2 控制单元，"转矩给定"模式 + VECTOR 多轴同步）  
- ABB ACS880（DriveSync 模式，主从速度/转矩跟随）  
- 安川 GA700（"软件主从"模式，模拟量或 MECHATROLINK 链路）

### 4.2 电子轴 / 虚拟主轴（Virtual Master Axis）

**原理：**  
在上位机（PLC/运动控制器）中以固定时钟节拍生成"虚拟主轴"速度曲线，每台变频器按电子齿轮比跟随该曲线。没有物理主机，所有从站地位等同。

```
上位机软件：以 1ms 节拍输出 v_master(t)
从站1：v1 = v_master × 齿轮比1
从站2：v2 = v_master × 齿轮比2
从站N：vN = v_master × 齿轮比N
```

**优点：**  
- 所有从站精确同步，无相位差。  
- 可实现凸轮（电子凸轮）、插补等复杂运动。

**缺点：**  
- 依赖高实时性通信总线（EtherCAT、PROFINET IRT）。  
- 驱动器需要支持位置/速度跟随模式（伺服级）。  
- 标准 Modbus RTU 带宽与延时无法满足需求。

**代表产品与案例：**  
- 倍福 TwinCAT 3（EtherCAT + NC/PTP 软轴，应用于印刷机、纸箱成型机）  
- 博世力士乐 IndraMotion for Packaging（电子轴，用于多列包装机）  
- 三菱 SSCNET III/H（光纤高速同步总线，应用于汽车焊装线）  

### 4.3 下垂控制（Droop Control / Speed Drooping）

**原理：**  
为每台变频器的频率给定引入一个与负载（电流）成正比的下垂系数，使高负载从站自动降速、低负载从站自动升速，实现负载均衡而不依赖高速通信。

```
f_i = f_set - k_droop × (I_i - I_rated)
```

其中 `k_droop` 为下垂系数（Hz/A），`I_i` 为第 i 台变频器的实际电流。

**优点：**  
- 各从站完全解耦，无通信依赖，天然抗通信故障。  
- 负载均衡自发达成，不需要上位机协调。

**缺点：**  
- 存在速度稳态误差（有差控制）。  
- 需要驱动器支持电流/转矩反馈输出。

**代表案例：**  
- 纸浆造纸机多电机段驱动（每段设置略有差异的基速与下垂特性）  
- 大型皮带输送机（多驱动点分布式驱动，西门子 S120 / ABB ACS880 应用）  

### 4.4 交叉耦合控制（Cross-Coupling Control）

**原理：**  
每台变频器运行独立 PID，同时引入一个"同步误差修正项"，使各从站速度向整体均值收敛：

```
对从站 i：
e_sync_i = v_i - v_avg
u_i = PID_i(e_speed_i) - k_sync × e_sync_i
```

**优点：**  
- 既保留各从站独立调速能力，又通过同步项约束速度离散度。  
- 经典控制理论有完整的稳定性分析。

**缺点：**  
- 需要实时计算各从站平均速度，通信要求中等。  
- 参数整定复杂（每台 Kp/Ki/Kd + 全局 k_sync）。

**代表案例：**  
- 多轴 CNC 进给轴同步（Y1/Y2 龙门双轴）  
- 高精度分条机双卷轴同步张力控制  
- 论文：Koren & Lo, "Variable-gain cross-coupling controller for contouring" (1991)

### 4.5 模型预测控制（Model Predictive Control，MPC）

**原理：**  
建立多从站系统的预测模型，在每个控制周期内求解使未来 N 步误差最小化的最优控制序列，考虑约束（转矩上下限、速度上下限）。

**优点：**  
- 可精确建模纯滞后，从根本上克服大死区问题。  
- 统一处理多输入多输出耦合。

**缺点：**  
- 计算量大（需在线求解二次规划），对嵌入式控制器要求高。  
- 建模难度高，被控对象参数变化时模型失配。

**代表产品：**  
- Rockwell Automation PlantPAx MPC  
- 西门子 SIMIT 动态仿真 + 优化控制器

### 4.6 变频器内置多机同步功能

部分变频器已内置多机同步，无需上位机介入：

| 厂商 | 产品 | 同步方案 |
|---|---|---|
| 西门子 | SINAMICS S120 | CU320-2 内部 VECTOR 同步，支持 1:N 主从 |
| ABB | ACS880 | DriveSync（驱动器间模拟量/数字量主从） |
| 施耐德 | Altivar 660 | CANopen/Modbus 主从跟随 |
| 安川 | GA500/GA700 | 模拟量速度跟随 + MECHATROLINK 同步 |
| 台达 | VFD-CP2000 | 内置 PLC + 多机通讯同步 |
| 雷码 LM1000H | — | 无内置多机同步；需上位机协调（本仓库方案） |

## 5. 当前架构分析与改进建议

### 5.1 现状总结

| 维度 | 当前状态 | 风险 |
|---|---|---|
| 通信拓扑 | Modbus RTU/TCP，串行轮询 N 从站 | 纯滞后随 N 线性增大 |
| PID 控制器 | 单 PID，聚合速度反馈 | 无法区分从站间负载差异 |
| 控制输出 | 全从站写相同 `P3.10` 转矩 | 无个体补偿能力 |
| 速度聚合 | Min/Avg/Median 策略可配置 | 各策略均引入信息损失或偏差 |
| 稳速判定 | 窗口内实时速度偏差 ≤ 容差 | 聚合速度稳定但从站间离散度可能仍大 |

### 5.2 阶段性改进建议（按复杂度升序）

#### 短期：优化现有架构（无需修改算法）

1. **降低 PID 增益，延长积分时间**  
   多从站场景的纯滞后是单从站的 N 倍，须对应等比例降低 Kp（参考 Cohen-Coon 整定法则）：  
   ```
   N 从站建议 Kp ≈ 单从站 Kp / N
   N 从站建议 Ki ≈ 单从站 Ki / N²
   ```

2. **启用速度扩散告警**  
   `SpeedSpreadTooLargeDetected` 事件已实现，建议配置 `stabilizedToleranceMmps` 小于从站间允许的最大速度差，及时发现从站间不同步。

3. **调整聚合策略**  
   - 从站负载差异较大时推荐 `Median`（抗极值干扰）  
   - 从站负载较均匀时可用 `Avg`（减小等效噪声方差）  
   - 避免使用 `Min`（会驱动轻载从站持续超速）

4. **增大写入节流间隔（`torqueSetpointWriteInterval`）**  
   减少不必要的转矩更新频率，降低通信总线负载，缓解 Modbus 拥塞对采样延时的影响。

#### 中期：引入主从转矩跟随（推荐，侵入最小）

在 `LeiMaLoopTrackManager` 中实现简化版主从转矩跟随：

1. 指定一个从站（如从站地址最小的）作为速度反馈主机。  
2. PID 闭环只使用主机的速度反馈。  
3. PID 输出的转矩给定广播给全部从站（包括主机）。  
4. 主机负责速度稳定，从机通过相同转矩自然跟随。

```
只用从站1的速度 → PID → 转矩T → 广播到从站1、2、3
从站2、3：以相同转矩运行，通过机械耦合自然同步
```

**优点：** 通信量不增加，代码改动集中在速度采样逻辑（只读主机）。  
**局限：** 如果皮带松紧不均导致从站间负载差异大，各从站速度仍会有偏差，但不会因 PID 二次放大。

#### 长期：交叉耦合或分布式 PID（需要通信架构升级）

如果对同步精度有严格要求（如相邻从站速度差 < 20 mm/s），需要：

1. 升级通信至 **Modbus TCP 并行轮询**（各从站 TCP 连接独立，并发请求），将采样时差从 N×T 降至 max(T)。  
2. 为每个从站维护独立 PID，输出独立转矩给定。  
3. 引入交叉耦合同步项，使各从站速度向均值收敛。

## 7. 小结

| 因素 | 单从站 | 多从站 |
|---|---|---|
| 采样时差 | 无 | N×T_slave（随从站数量线性增大） |
| 机械耦合 | 无 | 皮带张力强耦合，相互干扰 |
| 控制分辨率 | 单对单 | 一对多，无个体补偿 |
| PID 稳定裕度 | 大 | 小（纯滞后大、噪声强） |
| 参数整定难度 | 低 | 高（非线性、多变量） |

**工程界已验证的最优路径：** 对于不具备 EtherCAT/PROFINET 实时总线的 Modbus 场景，**主从转矩跟随**是复杂度最低、效果最好的折中方案；进一步提升同步精度的次优选择是**交叉耦合控制 + TCP 并行采样**。
