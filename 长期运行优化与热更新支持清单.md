# 长期运行优化与热更新支持清单

## 需要优化的内容

### 1. 启停后可生效、运行中不可热更新的配置收敛
- 问题：部分能力开关与厂商注册决策在 Host 启动阶段通过 `GetValue/GetSection().Get<T>()` 读取，运行中修改配置不会触发服务注册变更。
- 优化方向：
  - 将“启用/禁用”决策从启动时分支迁移到运行期判定（托管服务内部基于 `IOptionsMonitor<T>.CurrentValue` 决策）。
  - 对无法运行期切换的能力明确标注“需重启生效”。

### 2. 长期运行场景的日志治理增强
- 问题：虽然日志清理已修复为递归扫描，但长期运行仍需要可观测治理，避免磁盘容量风险。
- 优化方向：
  - 增加日志目录容量告警（目录大小、文件数量、磁盘剩余空间阈值）。
  - 为不同业务分类配置独立保留策略，并统一收敛到 `LogCleanupHostedService`。

### 3. 热路径持续压测与回归基线
- 问题：当前热路径已具备快速路径与缓存，但需建立“长期运行 + 高频事件”压测基线防止后续回归。
- 优化方向：
  - 固化分拣链路（创建/成熟/上车/落格）高频压测用例。
  - 监控关键指标：队列长度、事件处理延迟、落格吞吐、异常率。

## 当前不支持热更新的项（代码现状）

> 说明：以下项在代码中以启动期读取方式决定（`GetValue` / `GetSection().Get<T>()` / `IOptions<T>.Value` 构造单例），运行中改配置不会动态生效。

### A. 启动期服务注册分支（需重启生效）
- `LoopTrack:Enabled`、`LoopTrack:Hil:Enabled`  
  - 位置：`Zeye.NarrowBeltSorter.Host/Vendors/DependencyInjection/HostApplicationBuilderLoopTrackExtensions.cs`
  - 影响：决定注册 `LoopTrackManagerHostedService` 还是 `LoopTrackHILHostedService`。

- `Chutes:Enabled`、`Chutes:Vendor`、`Chutes:ZhiQian:Enabled`  
  - 位置：`Zeye.NarrowBeltSorter.Host/Vendors/DependencyInjection/HostApplicationBuilderZhiQianExtensions.cs`
  - 影响：决定是否注册智嵌格口管理器与相关托管服务。

- `Chutes:ForcedRotation:Enabled`、`Chutes:DropSimulation:Enabled`  
  - 位置：`Zeye.NarrowBeltSorter.Host/Vendors/DependencyInjection/HostApplicationBuilderZhiQianExtensions.cs`
  - 影响：决定是否注册 `ChuteForcedRotationHostedService` 与 `ChuteDropSimulationHostedService`。

- `Leadshaine:EmcConnection:Enabled`、`Leadshaine:IoLinkage:Enabled`、`Leadshaine:Sensor` 是否有绑定、`Leadshaine:SignalTower:Enabled`  
  - 位置：`Zeye.NarrowBeltSorter.Host/Vendors/DependencyInjection/HostApplicationBuilderLeadshaineExtensions.cs`
  - 影响：决定是否注册 IoMonitoring/IoPanelStateTransition/IoLinkage/CarrierLoopGrouping/SignalTower 等服务与实现。

- `Leadshaine:EmcConnection:Enabled` + 格口开关组合（分拣总编排注册条件）
  - 位置：`Zeye.NarrowBeltSorter.Host/Vendors/DependencyInjection/HostApplicationBuilderSortingExtensions.cs`
  - 影响：决定是否注册分拣任务主编排链路。

### B. 启动期构造单例读取的配置（运行中改值通常不生效）
- 通过 `IOptions<T>.Value` 注入构造的设备管理器/控制器配置（如 Leadshaine EMC/IoPanel/Sensor/SignalTower 相关）
  - 位置：`Zeye.NarrowBeltSorter.Host/Vendors/DependencyInjection/HostApplicationBuilderLeadshaineExtensions.cs`
  - 影响：对象构造后持有启动快照，后续配置变更不自动刷新到已创建实例。

## 当前支持热更新的典型项（用于对照）
- `LogCleanupSettings`（日志清理周期、保留天数、目录）  
  - 位置：`Zeye.NarrowBeltSorter.Execution/Services/LogCleanupHostedService.cs`
  - 原因：运行中通过 `IOptionsMonitor<LogCleanupSettings>.CurrentValue` 读取。

- `SortingTaskTimingOptions`（分拣时序参数）  
  - 位置：`Zeye.NarrowBeltSorter.Execution/Services/SortingTaskOrchestrationService.cs`、`SortingTaskDropOrchestrationService.cs`
  - 原因：运行中通过 `IOptionsMonitor<SortingTaskTimingOptions>.CurrentValue` 读取。

## 建议优先级
- P0：将“是否注册服务”的关键开关补充“需重启生效”文档，并在运行日志中打印一次当前生效模式。
- P1：将可运行期判定的开关从启动期分支迁移为服务内部 `IOptionsMonitor` 判定。
- P2：补齐长期运行压测与容量告警基线。
