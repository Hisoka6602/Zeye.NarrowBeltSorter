# 逐文件代码健康检查方案（多 PR 执行）

## 1. 目标与范围

- 目标：对仓库内代码与关键配置逐文件完成全覆盖检查，识别并记录以下问题：
  - 影分身代码（重复实现/复制粘贴）
  - 过度设计与冗余代码
  - 性能问题（热路径分配、不必要 I/O、低效集合操作等）
  - 逻辑问题（边界、状态流转、异常路径）
  - 并发与竞态问题（共享状态、订阅发布、取消与资源释放）
  - 违反 `.github/copilot-instructions.md` 的规则项
- 范围：
  - `Zeye.NarrowBeltSorter.Core`
  - `Zeye.NarrowBeltSorter.Drivers`
  - `Zeye.NarrowBeltSorter.Execution`
  - `Zeye.NarrowBeltSorter.Host`
  - `Zeye.NarrowBeltSorter.Core.Tests`
  - 根目录与 `.github` 下与规则校验相关的文档/脚本/工作流

## 2. 执行原则

- 仅检查与记录，不在本轮检查 PR 中直接改代码。
- 采用“逐文件打勾 + 证据留痕”机制，保证可追溯、可复核、无遗漏。
- 支持拆分多个 PR：每个 PR 负责一个或多个目录批次，最终汇总闭环。
- 任何发现项都必须绑定“文件路径 + 行号区间 + 问题分类 + 风险等级 + 建议动作”。

## 3. 逐文件检查清单模板

每个文件检查完成后，按以下模板记录：

- 文件：`<绝对或仓库相对路径>`
- 检查结论：`通过 / 有问题`
- 问题分类（可多选）：
  - 重复/冗余
  - 设计复杂度
  - 性能
  - 逻辑正确性
  - 并发安全
  - 规则合规（copilot-instructions）
- 证据：`行号区间 + 现象`
- 风险等级：`高/中/低`
- 建议动作：`修复 / 重构 / 保持现状（给出理由）`

## 4. 分批执行策略（建议）

### PR-A：Core + Host（规则与边界优先）

- Core：接口、枚举、事件载荷、Options、Utilities、Manager
- Host：依赖注入、配置绑定、配置注释完整性
- 重点：
  - 架构边界是否被侵入
  - 枚举/事件/Options/接口目录规则是否满足
  - 时间语义是否统一为本地时间（禁止国际标准时区语义链路）
  - SafeExecutor 使用是否统一

### PR-B：Drivers（设备协议与性能优先）

- Leadshaine / LeiMa / ZhiQian 全部逐文件检查
- 重点：
  - 协议实现是否与文档一致
  - 热路径是否存在冗余分配与重复读写
  - 异常路径是否全部记录日志并可落盘
  - 通信库使用是否符合强制约束（TouchSocket/TouchSocket.Modbus/Polly）

### PR-C：Execution + Tests（并发与编排优先）

- Services、Parcel、State、Hosted、测试桩与单测
- 重点：
  - 事件发布是否非阻塞且订阅者并行
  - 状态切换是否统一走 `ISystemStateManager.ChangeStateAsync`
  - 是否存在触发重放/缓冲重放等违规机制
  - 资源释放、取消令牌、后台循环退出是否完整

### PR-D：根目录文档与规则脚本（闭环）

- 根目录中文文档、README、结构清单、规则脚本
- 重点：
  - 文件树与职责说明是否与仓库一致
  - 两个结构清单是否保持树状图 + `#` 职责说明
  - 检查记录是否完整归档并可复审

## 5. 逐文件检查步骤（每个 PR 内统一执行）

1. 建立“待检文件总清单”，逐项编号。
2. 按编号逐文件检查并填写模板记录。
3. 对“有问题”文件生成问题条目，标注风险等级。
4. 完成本批次后执行一次遗漏复核：
   - 总文件数 = 已检查通过 + 已记录问题文件数。
5. 输出批次总结：
   - 本批次文件总数、通过数、问题数（按分类统计）。
6. 将问题列表转换为后续“修复 PR 待办池”。

## 6. 重点规则核对项（来自 copilot-instructions）

- 时间语义：禁止引入国际标准时区语义与转换链路，统一本地时间。
- 目录约束：枚举、Options、接口、静态工具类、事件载荷目录必须符合规定。
- 类型约束：事件载荷必须使用 `readonly record struct`。
- 日志约束：异常路径必须记录并落盘；仅允许 NLog；文件轮转上限 10MB。
- 状态约束：状态切换统一通过 `ISystemStateManager.ChangeStateAsync`；急停优先级最高。
- 并发约束：事件发布非阻塞、订阅者并行、禁止触发重放机制。
- 结构约束：单文件单类、避免重复实现、严格层级边界与低侵入。

## 7. 验收清单（检查 PR 通用）

- [ ] 已生成本批次逐文件检查清单，且文件总数可对账。
- [ ] 每个文件均有检查结论（通过/有问题），无空项。
- [ ] 所有问题均包含证据（文件 + 行号区间 + 分类 + 风险）。
- [ ] 已完成遗漏复核并给出一致性结论。
- [ ] 已输出后续修复 PR 待办池（按优先级）。

## 8. 交付物

- 本方案文档（用于统一检查口径）。
- 分批 PR 的逐文件检查记录。
- 问题清单与修复优先级建议。
